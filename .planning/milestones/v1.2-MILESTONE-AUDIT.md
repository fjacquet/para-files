## Post-Audit Bug Fix

A production regression was discovered during this audit session and fixed before archiving:

**Fix:** `RoutingRuleModel` in `reference_tree.py` had two over-strict Pydantic constraints introduced in Phase 10:
- `destination` was required (`Field(min_length=1)`) but action-based rules (e.g. `generic_folders` with `action: flatten_to_inbox`) have no destination field
- `platforms` was typed as `list[str]` but the actual YAML uses `dict[str, list[str]]` (platform-keyed glob lists)

Both caused every file classification to fail with `"Invalid reference tree YAML"` on real configs.

**Resolution:** `destination` made optional (`Field(default="")`), `platforms` widened to `dict[str, list[str]] | list[str] | None`. Tests updated. Committed before archiving.
