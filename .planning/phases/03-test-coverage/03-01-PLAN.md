<feature>
  <name>Pipeline classifier exception isolation</name>
  <files>tests/test_pipeline.py</files>
  <behavior>
    Pipeline.classify() must not crash when a classifier raises.
    It must skip the failing classifier and try the next one.
    If no classifier succeeds, it must return 0_Inbox.

    Cases:
    - [failing_classifier, passing_classifier] → passing_classifier result (not 0_Inbox)
    - [failing_classifier] → 0_Inbox (ClassificationSource.DEFAULT, confidence 0.0)
    - [failing_1, failing_2, failing_3] → 0_Inbox (no crash)
    - logger.exception called with classifier name when classifier fails

    Implementation pattern (inject mocks into pipeline._classifiers):

      pipeline = ClassificationPipeline(mock_config)
      pipeline._ensure_initialized()

      failing = MagicMock()
      failing.name = "failing_classifier"
      failing.classify.side_effect = RuntimeError("boom")

      passing = MagicMock()
      passing.name = "passing_classifier"
      passing.classify.return_value = ClassificationResult(
          category="3_Resources/test",
          confidence=Confidence(value=0.85, source=ClassificationSource.TAXONOMY_CLASSIFIER),
      )

      pipeline._classifiers = [failing, passing]  # Replace, not insert

    Use monkeypatch or unittest.mock.patch("loguru.logger.exception") to capture log calls.
  </behavior>
  <implementation>
    Add three test methods to the existing TestClassificationPipeline class in tests/test_pipeline.py.
    Do NOT modify production code.
    Import additions needed: from unittest.mock import call, patch (patch already imported)

    test_pipeline_skips_failing_classifier_and_continues:
      - Build pipeline, replace _classifiers with [failing_mock, passing_mock]
      - Call pipeline.classify("any content")
      - Assert result.category == "3_Resources/test" (passing_mock result)
      - Assert result.confidence.source == ClassificationSource.TAXONOMY_CLASSIFIER
      - Assert failing_mock.classify.call_count == 1 (was called once, raised)
      - Assert passing_mock.classify.call_count == 1 (was then called)

    test_pipeline_returns_default_when_only_classifier_fails:
      - Build pipeline, replace _classifiers with [failing_mock]
      - Call pipeline.classify("any content")
      - Assert result.category == "0_Inbox"
      - Assert result.confidence.source == ClassificationSource.DEFAULT
      - Assert result.confidence.value == 0.0

    test_pipeline_returns_default_when_all_classifiers_fail:
      - Build pipeline, replace _classifiers with [failing_1, failing_2, failing_3]
        where each has side_effect=ValueError("classifier N failed")
      - Call pipeline.classify("any content")
      - Assert result.category == "0_Inbox"
      - Assert result.confidence.source == ClassificationSource.DEFAULT
      - Assert all three classify methods called exactly once each
  </implementation>
</feature>
