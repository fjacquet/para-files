**TestInboxStats (unit tests for `_InboxStats` dataclass):**

**TestProcessInboxFile (unit tests for `_process_inbox_file`):**
Use `MagicMock` for pipeline. `_InboxStats(total=3)` passed in.

- `test_confidently_classified_file_is_moved` — pipeline returns `ClassificationResult` with `confidence.source = ClassificationSource.RULES_ENGINE` (not DEFAULT). Mock `move_classified_file` to return a success `MoveResult`. Assert `stats.moved == 1`, `stats.stayed == 0`, `stats.by_signal["rules_engine"] == 1`. Use `unittest.mock.patch("para_files.cli.inbox_cmd.move_classified_file")`.

- `test_default_classified_file_stays` — pipeline returns result with `confidence.source = ClassificationSource.DEFAULT`. Assert `stats.stayed == 1`, `stats.moved == 0`. `move_classified_file` must NOT be called.

- `test_pipeline_exception_increments_failed` — pipeline.classify_file raises `RuntimeError`. Assert `stats.failed == 1`, `stats.moved == 0`, `stats.stayed == 0`.

- `test_failed_move_increments_failed` — pipeline returns confident result, `move_classified_file` returns `MoveResult(success=False, ...)`. Assert `stats.failed == 1`, `stats.moved == 0`.

**TestPrintInboxSummary (unit tests for `_print_inbox_summary`):**
Use `typer.testing.CliRunner` or capture `typer.echo` via `capsys` from pytest.

- `test_summary_shows_all_counts` — `stats = _InboxStats(total=10, moved=7, stayed=2, failed=1, by_signal={"rules_engine": 5, "book_detector": 2})`. Call `_print_inbox_summary(stats, dry_run=False)`. Capture stdout via capsys. Assert output contains `"Total processed"`, `"10"`, `"Moved"`, `"7"`, `"Stayed in Inbox"`, `"2"`, `"Errors"`, `"1"`, `"rules_engine"`, `"5"`.

- `test_summary_dry_run_label` — `dry_run=True`. Assert output contains `"dry run"`.

- `test_summary_no_errors_line_when_zero` — `stats.failed == 0`. Assert `"Errors"` does NOT appear in output.

**TestInboxCommand (integration tests via CliRunner):**
Use `typer.testing.CliRunner` and `tmp_path` fixture. Mock `ClassificationPipeline` to avoid real MLX loads.

- `test_inbox_no_files` — empty temp directory. Run `inbox <dir>`. Assert exit_code == 0, output contains `"No files found"`.

- `test_inbox_dry_run_does_not_move_files` — create two dummy `.txt` files in `tmp_path`. Patch `ClassificationPipeline.classify_file` to return a confident result (RULES_ENGINE, 0.95). Patch `move_classified_file` to return success MoveResult. Run `inbox <dir> --dry-run`. Assert `move_classified_file` was called with `dry_run=True`. Assert files still present in `tmp_path` (they weren't actually moved because we patched the mover).

- `test_inbox_progress_output_format` — one file in tmp_path. Patch pipeline to return confident result. Patch mover to succeed. Run `inbox <dir>`. Assert output contains `"[1/1]"` (progress counter). Assert output contains the destination path from mock.

- `test_inbox_leaves_unclassifiable_files` — one file in tmp_path. Patch `classify_file` to return DEFAULT result. Run `inbox <dir>`. Assert `move_classified_file` NOT called. Assert output contains `"INBOX"` or `"stayed"` in summary.

- `test_inbox_summary_by_signal` — two files, both classified by RULES_ENGINE. Run `inbox <dir>`. Assert summary output contains `"rules_engine"` and `"2"`.

Each test must be independent. Use `unittest.mock.patch` as context managers or decorators. Construct `ClassificationResult` and `Confidence` objects directly from `para_files.types` — do not mock the types themselves.

For `MoveResult`, construct it as: `MoveResult(source=file_path, destination=target/file.txt, action="moved", success=True, message=None)`. Check `from para_files.mover import MoveResult` for the actual field names by reading the module if needed.
